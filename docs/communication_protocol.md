# 串口通信协议文档

## 1. 概述

本文档定义 Linux 侧与 RT-Thread 侧之间的串口通信协议。

- **物理接口**：UART（串口）
- **波特率**：115200
- **数据位**：8
- **停止位**：1
- **校验位**：无
- **数据流向**：全双工

## 2. 帧格式

| 字段 | 字节数 | 说明 |
|------|--------|------|
| 帧头 | 1 | 固定 0xAA |
| 数据长度 | 1 | 数据字段字节数 |
| 指令类型 | 1 | 功能码 |
| 数据 | 0~8 | 具体数据 |
| 校验和 | 1 | 异或校验 |

## 3. 指令类型

| 指令码 | 方向 | 名称 | 说明 |
|--------|------|------|------|
| 0x01 | Linux→RT-Thread | 速度控制 | vx, vy, omega |
| 0x02 | Linux→RT-Thread | 急停 | 立即停止 |
| 0x03 | RT-Thread→Linux | 里程计反馈 | x, y, theta |
| 0x04 | RT-Thread→Linux | 传感器状态 | 编码器/IMU |

## 4. 指令详细定义

### 4.1 速度控制 (0x01)

数据字段 6 字节（int16小端）：

| 字节 | 内容 | 单位 | 范围 |
|------|------|------|------|
| 0-1 | vx | mm/s | -1000~1000 |
| 2-3 | vy | mm/s | -1000~1000 |
| 4-5 | omega | 0.01 rad/s | -500~500 |

示例：vx=500mm/s, vy=0, omega=0  
`AA 06 01 F4 01 00 00 00 00 [校验]`

### 4.2 急停 (0x02)

数据字段为空（长度=0）。  
示例：`AA 00 02 [校验]`

### 4.3 里程计反馈 (0x03)

数据字段 12 字节（int32小端）：

| 字节 | 内容 | 单位 |
|------|------|------|
| 0-3 | x | mm |
| 4-7 | y | mm |
| 8-11 | theta | 0.01 rad |

### 4.4 传感器状态 (0x04)

数据字段 8 字节：

| 字节 | 内容 |
|------|------|
| 0-3 | 左前轮编码器脉冲数 |
| 4-7 | IMU俯仰角（0.01°） |

## 5. 校验和计算

异或：从帧头到数据字段最后一个字节。

C语言：
```c
uint8_t calc_checksum(uint8_t *buf, uint8_t len) {
    uint8_t cs = 0;
    for (int i = 0; i < len; i++) cs ^= buf[i];
    return cs;
}
```

python：
```python
def calc_checksum(data: bytes) -> int:{
    cs = 0
    for b in data:
        cs ^= b
    return cs；
}
```
